# 海面光电监视设备 cam_mon_cpp 设计文档

## 1. 概述

本模块是海面光电监视设备的C/C++通信库，实现了与跟踪处理器的UDP网络通信协议（协议版本V0.05）。主要负责：
- 向跟踪处理器下发控制命令（可见光摄像机、红外热像仪、跟踪处理器、环控设备、扫描方式等）
- 接收跟踪处理器上报的状态信息（负载状态、识别信息、跟踪状态等）
- 提供JNI接口供Java层调用

## 2. 协议规范

### 2.1 通信参数

| 项目 | 值 | 说明 |
|------|-----|------|
| 协议类型 | UDP | 控制指令 |
| 单播地址 | 193.0.1.94 | 跟踪处理器IP |
| 单播端口 | 8080 | 控制指令端口 |
| 组播地址 | 239.255.43.21 | 指令回传组播地址（V0.05） |
| 组播端口 | 23232 | 指令回传端口（V0.05） |

### 2.2 报文格式

```
+--------+--------+--------+--------+--------+--------+--------+
| 头部区 |        数据区        |  尾部区  |
|--------+--------+--------+--------+--------+--------+--------|
| 帧头(2B)| 地址位 | 功能位 | 控制位 | 数据位 | 校验位 | 帧尾(2B) |
| 0xFF0F | 1字节  | 1字节  | 1字节  | N字节  | 1字节  | 0xF00F  |
+--------+--------+--------+--------+--------+--------+--------+
```

**注**：多字节采用低字节在前，高字节在后的方式。

### 2.3 设备地址定义

| 设备名称 | 地址字节 | 说明 |
|----------|----------|------|
| 可见光跟踪处理器 | 0x00 | 直接连接可见光相机 |
| 红外跟踪处理器 | 0x01 | 直接连接红外热像仪 |
| 可见光摄像机 | 0x02 | 通过可见光跟踪处理器转发 |
| 红外热像仪 | 0x04 | 通过红外跟踪处理器转发（V0.05修正） |
| 环控设备 | 0x04 | 环控设备 |
| 扫描方式 | 0x06 | 扫描生成器（V0.05修正） |
| OSD实时信息 | 0x07 | OSD实时信息显示 |

### 2.4 校验方式

- 普通命令：地址位到数据位所有字节的和校验（取低位1字节）
- 伺服控制命令：从帧头到预留备份字节的异或校验（XOR）

## 4. 核心数据结构

### 4.1 负载状态上报（TrackProcessorStatusReport）

总帧长49字节，数据区41字节：

| 字节偏移 | 字段名 | 长度 | 说明 |
|----------|--------|------|------|
| 5 | 伺服工作模式 | 1字节 | Bit3-0:伺服模式, Bit5-4:跟踪源, Bit7-6:自动模式 |
| 6-9 | 红外焦距 | 4字节 | float型 |
| 10-13 | 白光焦距 | 4字节 | float型 |
| 14-17 | 方位脱靶量 | 4字节 | float型 |
| 18-21 | 俯仰脱靶量 | 4字节 | float型 |
| 22-25 | 环控设备 | 4字节 | 01**02**格式 |
| 26-29 | 伺服方位角 | 4字节 | float型 |
| 30-33 | 伺服俯仰角 | 4字节 | float型 |
| 34 | 自检状态 | 1字节 | Bit0-7:各子系统状态 |
| 35 | 可见光图像状态 | 1字节 | 00:有图像, 01:无图像 |
| 36 | 可见光通信状态 | 1字节 | 00:有通信, 01:无通信 |
| 37 | 红外图像状态 | 1字节 | 00:有图像, 01:无图像 |
| 38 | 红外通信状态 | 1字节 | 00:有通信, 01:无通信 |
| 39 | 伺服故障码 | 1字节 | 0:正常 |
| 40 | 电子透雾状态 | 1字节 | 00:无效, 01:有效 |
| 41 | 雨刷状态 | 1字节 | 0:工作中, 1:空闲 |
| 42 | 加热状态 | 1字节 | 0:加热中, 1:空闲 |
| 43 | 俯仰伺服故障码 | 1字节 | 0:正常 |
| 44-45 | 记忆跟踪时间 | 2字节 | uint16_t, 0-5000ms |

### 4.2 识别上报（DetectReport）

总帧长51字节，地址位固定0x02：

| 字节偏移 | 字段名 | 长度 | 说明 |
|----------|--------|------|------|
| 3 | 识别通道 | 1字节 | 0x01:可见光, 0x02:红外 |
| 12-17 | 跟踪目标 | 6字节 | 目标类型(1B)+置信度(1B)+x脱靶量(2B)+y脱靶量(2B) |
| 18-47 | 识别/检测目标 | 6字节×5 | 最多5个目标 |

目标类型位定义：Bit0:无人机, Bit1:鸟类

### 4.3 跟踪上报（TrackReport）

总帧长22字节，地址位固定0x03：

| 字节偏移 | 字段名 | 长度 | 说明 |
|----------|--------|------|------|
| 3 | 识别通道 | 1字节 | 0x01:可见光, 0x02:红外 |
| 12 | 跟踪状态 | 1字节 | 0x00:未跟踪, 0x01:正常跟踪, 0x10:记忆跟踪 |
| 13 | 识别/检测状态 | 1字节 | 0x00:关, 0x01:检测有效, 0x02:识别有效 |

### 4.4 OSD实时信息（OsdRealtimeReport）

总帧长27字节，数据区25字节：

| 字节偏移 | 字段名 | 长度 | 说明 |
|----------|--------|------|------|
| 3 | 行号 | 1字节 | 00-04:第一至五行 |
| 4-28 | 显示内容 | 25字节 | ASCII码字符串，左对齐，受OSD显示指令统一控制 |

## 9. 修复记录 (2026-06-24)

- 问题描述：
  - test_ptz_control 输入的 Z 值（变焦/焦距）无效，P/T 有响应但 Z 无反应。
  - 原因：CamAJFLib::set_ptz(...) 仅构建并发送舵机（伺服）控制包，忽略了传入的 zoom 参数；同时 CamAJFLib::set_focus(...) 报文编码不符合设备示例（未按 float 小端写入、使用了不匹配的 function/ctrl）。
- 变更文件：
  - cam_mon_cpp/src/cam_ajf_lib.cpp
- 具体修改：
  - set_ptz(float azimuth, float elevation, float zoom, ...)：在发送伺服包后调用 set_focus(zoom)（将传入的 zoom 视为焦距，单位 mm），确保 Z 值会下发到摄像机。
  - set_focus(float focus)：按协议示例构造“焦距直达”命令，使用 function=0x06、ctrl=0x00，并将 float 按小端字节序写入 payload[0..3]（与示例报文 0F F0 02 06 00 00 00 C8 42 ... 对齐）。
- 验证：
  - 已在本机构建并运行 test_ptz_control（./test_ptz_control -H 193.0.1.94 -p 8080 -P 0 -T 0 -Z 100 -v）。
  - 程序成功发送舵机控制包和焦距直达包，但未收到设备的状态回包（“未获取到有效的 PTZ 状态数据”）。需在设备/网络侧进一步确认设备是否在线或是否需要不同 control/func 配置。
- 后续建议：
  - 使用抓包工具（tcpdump/wireshark）在本机网络接口捕获发送的 UDP 报文，确认报文与设备示例字节完全一致；如需，我可以代为抓包并回显十六进制报文以便对照。
  - 若设备要求不同的 control/func 或数据偏移，可基于抓包或设备文档进一步调整实现。
